## Task Isolation

Use one branch or worktree per substantial project area, especially when tasks
run on different servers:

```text
CPU-push integration
GPU-pull exploration
memory backpressure and join
GDS and I/O benchmarking
MPI multi-BD/GPU scheduling
new detector support
```

The shared `psana2-gpu` skill carries architecture contracts, not task progress.
Record task-specific state in commits, repository notes, or a handoff file on
the task branch.

Before editing:

```bash
git status --short --branch
git log -8 --oneline
git diff --stat origin/features/psana2-gpu...HEAD
```

Do not overwrite unrelated dirty files or resurrect code deleted by cleanup
commits.

## Required Source Audit

Trace the current call path rather than relying only on historical notes:

```text
DataSource(gpu_det=...)
  -> RunSerial or RunParallel
  -> GpuEvents
  -> KvikioGpuReader
  -> EventPool
  -> GPUDetector.process_batch()
  -> GpuEventContext / GPUResult
```

For EventBuilder or MPI changes, also trace:

```text
SmdReaderManager
EventBuilderManager / EventBuilder
Smd0 / EventBuilderNode / BigDataNode
EventManager
transition handling
```

Use `git ls-files psana/psana/gpu` to distinguish tracked source from stale
bytecode generated by older branches.

## Correctness Baseline

The consolidated automated suite intentionally focuses on:

```text
psana/psana/tests/gpu/unit/test_core.py
psana/psana/tests/gpu/integration/test_pixel_exact.py
```

The known acceptance baseline included:

- CPU-only invariant tests.
- A100 pixel-exact Jungfrau comparison on public data.
- Single-event processing.
- Batched slot reuse.
- Partial-tail handling.
- L1Accept segment order differing from sorted Configure order.
- BeginStep drain before calibration constants update.
- No unnecessary drain for ordinary transitions.
- EndRun drain exactly once.

Verify datasets and environment before GPU integration tests. KvikIO CPU
fallback validates correctness and lifetime but not true-GDS performance.

## Memory And Lifetime Tests

Backpressure changes need tests proving:

- Byte estimates match allocation requirements within documented margin.
- Buffer growth cannot exceed the configured quota.
- Slot reuse waits for downstream CUDA or D2H completion tokens.
- Advancing a Python generator alone does not release an async consumer.
- One EB batch can split into ordered GPU subbatches.
- Variable-sized and missing descriptors preserve event identity.
- Partial joins flush on BeginStep, EndRun, end-of-input, and `max_events`.
- One oversized event fails explicitly or invokes the configured fallback.
- Multiple BDs cannot exceed the aggregate GPU allocation policy.

Avoid tests that only assert `cp.get_default_memory_pool().used_bytes()` falls;
that does not account for all owners or physical CUDA allocation behavior.

## Performance Validation

Performance remains script-driven. Do not make hardware throughput thresholds
pytest requirements.

For a performance change:

1. Run correctness tests first.
2. Warm kernel compilation.
3. Record I/O path and GPU pinning.
4. Compare one BD/GPU before scaling.
5. Capture loop time, stage timing, memory high-water, and NIC traffic.
6. Run enough events to reduce startup/cache effects.
7. Separate load time from event-loop time.
8. Save commands and logs with the report.

Use this structure when moving work between Codex tasks or colleagues:

```text
Objective:

Base and head commits:
Dirty files intentionally left out:

Implemented:
Current authoritative call path:
Architecture invariants affected:

Memory owners and lifetime:

```

State explicitly whether each claim is implemented, proposed, or measured.
